# MetricCanvas(指标画布)

一套以业务指标为核心的 AI 原生数据分析与可视化平台:数据服务提供数据,声明式页面描述应用,统一运行时渲染页面,AI 完成页面组合与数据分析。

## Language

**数据服务 (Data Service)**:
建设在 GraphQL 之上的统一数据查询服务,提供表、指标、维度和聚合查询能力,返回标准化格式。本项目的唯一数据入口。
_Avoid_: 表服务(旧称,2026-07-20 更名)、数据接口、查询服务、API 层

**数据服务仿真 (Data Service Sim)**:
开发期在本地按数据服务的真实协议(apiQuery 信封、方言指令、元数据端点、错误码)提供数据的仿真服务,种子数据入库、方言真执行。与 mock 的区别:mock 绕过协议直接向数据网关端口供数,仿真让真实适配器走真实 HTTP 边界;它是《中间层分析.md》所记协议假设的可执行文档,真实联调发现分歧时以真实服务为准修订仿真。
_Avoid_: 假后端、本地 mock 服务

**数据网关 (Data Gateway)**:
统一运行时的取数端口:生效查询进、数据行出。按意图命名,不感知数据来自谁;数据服务是其一期唯一真实适配器,mock 是开发期适配器。新增数据源须经词汇表与 ADR 决策,不得绕过网关私连。
_Avoid_: 数据源封装、fetcher

**看板页面 (Dashboard Page)**:
平台的聚合根与核心资产:以指标为核心、可交互的看板单元。以严格声明式文档形式存在(一期存 Git),由开发者编写或 AI 生成,可版本管理、可 diff、可评审;同一性由页面 id 承载,运行时负责将其呈现为可见界面。需要指称其文档形态时用普通词"页面文档"。
_Avoid_: 页面规格(旧称,2026-07-20 降级为文档形态,ADR-0007)、页面配置、JSON 配置、大屏

**领域 DSL (Domain DSL)**:
看板页面文档所使用的语言,以指标、维度、聚合、筛选联动为一等公民,自主定义并演进,不绑定外部 UI 协议。

**统一运行时 (Runtime)**:
读取看板页面并将其渲染为可交互界面的引擎。页面只描述"是什么",渲染行为全部由运行时决定。
_Avoid_: 渲染器、前端框架

**结构化查询 (Structured Query)**:
看板页面中声明数据需求的形态:指标+维度+筛选+聚合组成的 JSON 对象,由运行时翻译为数据服务请求。页面中不出现查询语句字符串。
_Avoid_: SQL、查询语句、DSL 查询

**渲染通道 (Render Channel)**:
消费同一个看板页面并将其变成可见界面的一条路径。当前唯一通道是统一运行时;未来 A2UI 适配器是第二条通道。页面不感知通道的存在。

**纯渲染组件 (Pure Component)**:
运行时组件集中的组件形态:只通过 props 接收数据快照与展示配置,只通过事件上抛交互;不发请求、不碰全局状态、不管理加载态。
_Avoid_: 智能组件、容器组件

**筛选状态 (Filter State)**:
页面级共享的筛选条件集合(维度值、时间范围)。筛选器写入它,图表声明订阅它;图表点击可回写它以实现联动下钻。联动只通过筛选状态传递,组件间不直接连线。

**跨页下钻 (Drill-through)**:
从一个页面的图表交互跳转到另一个页面并携带筛选条件的导航行为,在页面文档中以 `navigate` 交互声明,物理载体是 URL。区别于页内下钻(回写筛选状态)。

**生效查询 (Effective Query)**:
widget 声明的结构化查询与其订阅的筛选器当前值合成后的最终查询,是真正发往数据服务的查询。页面声明"意图",生效查询是"意图 × 当前筛选状态"的产物。

**数据快照 (Data Snapshot)**:
查询编排后分发给组件的数据包,含数据行与查询状态(就绪/加载/错误/空)。组件只认快照,不感知查询过程。
_Avoid_: 查询结果、响应数据

**指标 (Metric)**:
由数据服务定义和治理的业务度量。页面只引用指标,不定义指标的计算逻辑。

**元数据快照 (Catalog Snapshot)**:
数据服务指标/维度目录的版本化副本(供给侧清单),提交在页面仓库中,由 `sync-catalog` 命令同步。只含指标定义,零数据行;业务数据永远实时查数据服务、不落盘。
_Avoid_: mock 数据(mock 是开发期按快照形状造的假数,二者不同)

**指标需求清单 (Indicator Manifest)**:
需求讨论/数据分析阶段产出的需求侧指标清单:页面要什么指标、含义是什么、呈现在哪。由独立流程消费以寻找数据源并落成数据服务指标。与元数据快照(供给侧)、看板页面(消费侧)构成三方结构,`METRIC_GAP` 即需求与供给的差集。
_Avoid_: 指标文档、indicator 文件

**维度 (Dimension)**:
由数据服务定义的分析切分角度。页面只引用维度。
