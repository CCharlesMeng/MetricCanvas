# MetricCanvas(指标画布)

一套以业务指标为核心的 AI 原生数据分析与可视化平台:数据服务提供数据,声明式页面描述应用,统一运行时渲染页面,AI 完成页面组合与数据分析。

## Language

**数据服务 (Data Service)**:
建设在 GraphQL 之上的统一数据查询服务,提供表、指标、维度和聚合查询能力,返回标准化格式。本项目的唯一数据入口。
_Avoid_: 表服务(旧称,2026-07-20 更名)、数据接口、查询服务、API 层

**数据服务仿真 (Data Service Sim)**:
开发期在本地按数据服务的真实协议(apiQuery 信封、方言指令、元数据端点、错误码)提供数据的仿真服务,种子数据入库、方言真执行。与 mock 的区别:mock 绕过协议直接向数据网关端口供数,仿真让真实适配器走真实 HTTP 边界;它是《中间层分析.md》所记协议假设的可执行文档,真实联调发现分歧时以真实服务为准修订仿真。
_Avoid_: 假后端、本地 mock 服务

**数据网关 (Data Gateway)**:
统一运行时的取数端口:生效查询进、数据行出。按意图命名,不感知数据来自谁;数据服务是其一期唯一真实适配器,mock 是开发期适配器。新增真实数据入口须经词汇表与 ADR 决策,不得绕过网关私连。
_Avoid_: 数据源封装、fetcher

**看板页面 (Dashboard Page)**:
平台的聚合根与核心资产:以指标为核心的声明式数据应用单元。页面文档公开声明命名页面数据源、字段契约、嵌套内容分区、组件数据槽和有限交互(一期存 Git),由开发者编写或 AI 生成,可版本管理、可 diff、可评审;同一性由页面 id 承载,统一运行时负责取数并将其呈现为可见界面。需要指称其文档形态时用普通词"页面文档"。
_Avoid_: 页面规格(旧称,2026-07-20 降级为文档形态,ADR-0007)、页面配置、JSON 配置、大屏

**页面修订 (Page Revision)**:
同一看板页面在一次成功保存时形成的、经过校验且不可变的完整状态。修订沿线性历史前进,与页面文档的 `schemaVersion` 无关。
_Avoid_: 页面版本(易与 `schemaVersion` 混淆)、草稿快照

**发布租约 (Publish Lease)**:
人工确认发布期间对看板页面当前最新修订的限时独占权。租约绑定页面与修订,活动期间页面不能产生新修订。
_Avoid_: 发布锁、审批锁

**页面模板 (Page Template)**:
供搭建新看板页面复用的受治理起点,有独立身份、修订、权限与发布状态;其内容引用一个已发布页面修订,不复制页面文档。
_Avoid_: 页面副本、JSON 模板

**模板修订 (Template Revision)**:
同一页面模板在一次成功保存时形成的不可变状态,包含模板的发现元数据以及所引用的精确已发布页面修订;来源看板页面后续变化不改变既有模板修订。
_Avoid_: 模板版本、页面副本

**指标需求队列 (Metric Request Queue)**:
管理页面生成过程中经用户确认登记的指标缺口及其处理状态的队列。它不替代外部流程产出的指标需求清单。
_Avoid_: 指标需求清单、缺口日志

**目录隐藏 (Hidden from Catalog)**:
看板页面不再出现在目录发现结果中的展示状态,不是归档或删除;直接 URL 仍按权限访问当前发布修订。
_Avoid_: 归档、下线、删除

**领域 DSL (Domain DSL)**:
看板页面文档所使用的语言,以页面数据源、字段、指标、维度、聚合和筛选联动为一等公民,自主定义并演进,不绑定外部 UI 协议。

**页面数据源 (Page Data Source)**:
看板页面中的命名数据集及其输出契约,由 `fields` 和 `source` 组成。`source.type` 为 `inline` 时,数据行随页面固化,适合固定时点报告和离线交付;为 `query` 时,结构化查询由统一运行时经数据网关取数。组件只通过命名数据槽引用页面数据源,不携带查询或数据行;同一字段契约可在两种来源间切换。
_Avoid_: 渲染输入、快照信封、组件查询、数据源封装

**统一运行时 (Runtime)**:
读取看板页面,解析页面数据源并将其渲染为界面的引擎。页面只描述"是什么",取数编排、状态管理和渲染行为全部由运行时决定。
_Avoid_: 渲染器、前端框架

**结构化查询 (Structured Query)**:
`query` 页面数据源声明数据需求的形态:指标+维度+筛选+聚合组成的 JSON 对象,由统一运行时翻译为数据服务请求。组件不携带结构化查询,页面中不出现查询语句字符串。
_Avoid_: SQL、查询语句、DSL 查询

**渲染通道 (Render Channel)**:
消费同一个看板页面并将其变成可见界面的一条路径。当前唯一通道是统一运行时;未来 A2UI 适配器是第二条通道。页面不感知通道的存在。

**纯渲染组件 (Pure Component)**:
运行时组件集中的组件形态:只通过 props 接收已解析的数据行、字段契约与展示配置,只通过事件上抛交互;不发请求、不碰全局状态、不管理加载态。
_Avoid_: 智能组件、容器组件

**筛选状态 (Filter State)**:
页面级共享的筛选条件集合(维度值、时间范围)。筛选器写入它,query 页面数据源声明订阅它;组件 action 可回写它以实现联动下钻。联动只通过筛选状态传递,组件间不直接连线。

**跨页下钻 (Drill-through)**:
从一个页面的组件 action 跳转到另一个页面并携带筛选条件的导航行为,在页面文档中以 `props.actions[].navigate` 声明,物理载体是 URL。区别于页内下钻(回写筛选状态)。

**生效查询 (Effective Query)**:
`query` 页面数据源声明的结构化查询与其订阅的筛选器当前值合成后的最终查询,是真正发往数据服务的查询。页面声明"意图",生效查询是"意图 × 当前筛选状态"的产物。

**数据快照 (Data Snapshot)**:
统一运行时按组件数据槽分发的数据包,含数据行与数据状态(就绪/加载/错误/空)。`inline` 页面数据源同步产生终态数据快照,`query` 页面数据源经查询编排产生动态数据快照;组件不感知来源和取数过程。
_Avoid_: 查询结果、响应数据

**指标 (Metric)**:
由数据服务定义和治理的业务度量。页面只引用指标,不定义指标的计算逻辑。

**元数据快照 (Catalog Snapshot)**:
数据服务指标/维度目录的版本化副本(供给侧清单),提交在页面仓库中,由 `sync-catalog` 命令同步。只含指标定义,零数据行;`query` 页面数据源的业务数据经数据服务取得,固定时点数据可作为 `inline` 页面数据源的数据行进入看板页面,二者都不进入元数据快照。
_Avoid_: mock 数据(mock 是开发期按快照形状造的假数,二者不同)

**指标需求清单 (Indicator Manifest)**:
需求讨论/数据分析阶段产出的需求侧指标清单:页面要什么指标、含义是什么、呈现在哪。由独立流程消费以寻找数据来源并落成数据服务指标。与元数据快照(供给侧)、看板页面(消费侧)构成三方结构,`METRIC_GAP` 即需求与供给的差集。
_Avoid_: 指标文档、indicator 文件

**维度 (Dimension)**:
由数据服务定义的分析切分角度。页面只引用维度。候选值可含 '/' 表示层级(如 '华东/上海',树选筛选器按此建层级)——一期分隔符约定,数据服务供给层级元数据后替换。
