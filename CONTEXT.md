# MetricCanvas(指标画布)

一套以业务指标为核心的 AI 原生数据分析与可视化平台:表服务提供数据,页面规格描述应用,统一运行时渲染页面,AI 完成页面组合与数据分析。

## Language

**表服务 (Table Service)**:
建设在 GraphQL 之上的统一数据查询服务,提供表、指标、维度和聚合查询能力,返回标准化格式。本项目的唯一数据入口。
_Avoid_: 数据接口、查询服务、API 层

**页面规格 (Page Spec)**:
描述一个看板页面的结构化声明式文档,是平台的核心资产。由开发者编写或 AI 生成,可版本管理。
_Avoid_: 页面配置、JSON 配置、schema(schema 指规格的类型定义,不指规格实例)

**领域 DSL (Domain DSL)**:
页面规格所使用的语言,以指标、维度、聚合、筛选联动为一等公民,自主定义并演进,不绑定外部 UI 协议。

**统一运行时 (Runtime)**:
读取页面规格并渲染为可交互页面的引擎。页面规格只描述"是什么",渲染行为全部由运行时决定。
_Avoid_: 渲染器、前端框架

**结构化查询 (Structured Query)**:
页面规格中声明数据需求的形态:指标+维度+筛选+聚合组成的 JSON 对象,由运行时翻译为表服务请求。规格中不出现查询语句字符串。
_Avoid_: SQL、查询语句、DSL 查询

**渲染通道 (Render Channel)**:
消费同一份页面规格并将其变成可见界面的一条路径。当前唯一通道是统一运行时;未来 A2UI 适配器是第二条通道。规格不感知通道的存在。

**纯渲染组件 (Pure Component)**:
运行时组件集中的组件形态:只通过 props 接收数据快照与展示配置,只通过事件上抛交互;不发请求、不碰全局状态、不管理加载态。
_Avoid_: 智能组件、容器组件

**筛选状态 (Filter State)**:
页面级共享的筛选条件集合(维度值、时间范围)。筛选器写入它,图表声明订阅它;图表点击可回写它以实现联动下钻。联动只通过筛选状态传递,组件间不直接连线。

**指标 (Metric)**:
由表服务定义和治理的业务度量。页面规格只引用指标,不定义指标的计算逻辑。

**维度 (Dimension)**:
由表服务定义的分析切分角度。页面规格只引用维度。
