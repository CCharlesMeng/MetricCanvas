# 一期规格存 Git,平台后端与可视化管理延后

一期不建平台后端:页面规格以文件形式存 Git 仓库,走 PR 评审 + CI 校验 + 静态部署发布。长期演进为平台后端 + 数据库存储,提供版本历史、发布状态、权限 API 和可视化管理界面,为 AI 生成与在线编辑铺路。

一期读者看到"看板平台没有后端"会觉得奇怪,故记录:这是刻意的交付排序——先验证 DSL 与运行时这块地基,规格治理暂时复用 Git 的成熟能力,避免在 schema 未稳定时就固化数据库模型和管理界面。

## Consequences

- 运行时一期直接加载静态规格文件;规格加载须从第一天起走统一的抽象接口,后续切换到平台 API 时运行时不改。
- 二期入库时需要一次规格资产迁移(文件 → 数据库),成本可控。
