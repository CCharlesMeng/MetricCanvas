# "页面规格"降级为文档形态,领域词汇只保留聚合根"看板页面"

词汇表原有一等术语"页面规格 (Page Spec)",且 ADR-0006 以"页面是聚合根,规格是它的唯一真源"陈述二者关系。本决策将"规格"降级:领域中只有聚合根**看板页面**,"规格"是页面的序列化/存储形态,属于边界之下的实现细节;需要指称文档形态时用普通词"页面文档",不占术语地位。

理由:领域概念应映射现实——被消费、被评审、被 AI 生成的对象是页面本身;"可 diff 可校验"是页面以严格声明式文档存于 Git 带来的性质,由存储决策(ADR-0004)与格式约束(ADR-0003)表达即可,不需要介质词占据领域词汇。原有类型也暴露了术语含混:校验器实际接收的是不可信 unknown 文档,类型却宣称自己是"规格"。降级后边界更诚实:**文档进(加载),页面出(通过校验)**。

命名级联(代码):

- 聚合根类型 `PageSpec` → `Page`;
- 端口 `SpecProvider` → `PageRepository`(DDD 仓储:按 id 取聚合根;一期实现静态文件、二期平台 API,恰为仓储模式的教科书场景);
- `pageSpecSchema` → `pageSchema`;`specs/` 目录 → `pages/`;
- 文档格式版本字段 `specVersion` → `formatVersion`(破坏性变更,趁仓库仅一个页面文件时完成;N/N-1 兼容策略自 `formatVersion` 起算)。

## Considered Options

- **维持"页面规格"为一等术语**(原表述):证据充分——北极星句"以页面规格描述应用"、solution.md"核心资产从页面源代码转变为页面规格"、治理动词(评审/校验/生成/版本管理)的宾语皆为规格。被否决:这些陈述改写为"页面"同样成立且更自然("评审页面变更""AI 生成页面"),说明"规格"并非不可删的领域词;Terraform 类比支持元领域文档可以成为一等概念,但不强制。
- **双术语并存**("看板页面"+"页面规格",声明真源关系):同一聚合两个名字,文档与代码容易各说各话,违背一物一词的统一语言原则,被否决。

## Consequences

- 词汇表删"页面规格"、增"看板页面";活文档(solution.md、README、PRD、开放 issue)措辞级联,"规格即代码"改写为"页面即代码"。历史文档(origin.md、ADR-0001/0003/0004/0006、调研报告)保留原文,先例同"表服务"更名。
- ADR-0006 的分层与包命名决策不变(包名 `page` 反而更贴切);其中"页面规格是聚合根唯一真源"的表述由本 ADR 取代。
- 将来平台后端入库(二期),数据库存的是页面文档、运行时内存中是 `Page` 聚合,指称差异用"页面文档"这个普通词即可,不再新造术语。
