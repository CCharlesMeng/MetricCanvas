# 看板页面使用不可变线性修订与限时发布租约

看板页面保持稳定身份,每次成功保存产生一个经过当前元数据校验、内容不可变的页面修订;页面分别指向最新修订与当前发布修订。修订历史只允许以前一最新修订为基线线性前进,不采用覆盖写、分叉或事件溯源。

发布只能针对当前最新修订发起,并原子取得 15 分钟页面级发布租约。租约绑定页面、修订与发起人,活动期间禁止保存;人工确认时按最新元数据复验,成功后前移当前发布修订指针,批准、拒绝、取消或超时均释放租约。确认 URL 是单次入口而非授权凭据,所有写入口从首次提供起支持幂等重试。

## Considered Options

- 覆盖保存单份页面文档:接口简单,但无法精确预览、审计或安全回滚,被否决。
- 允许修订分叉或采用事件溯源:表达力更强,但二期只需要线性变更与显式发布记录,会把合并和投影复杂度提前引入,被否决。
- 确认期间继续接受保存并在批准时提示差异:评审看到的内容可能不再是发布内容,被否决。

## Consequences

- 预览始终绑定精确页面修订,稳定页面 URL 始终解析当前发布修订。
- 回滚不倒拨历史,而是复制旧内容形成新的最新修订后重新走发布流程。
- 保存、申请发布与确认发布必须在同一持久化事务内维护基线、租约、幂等和页面指针不变式。
